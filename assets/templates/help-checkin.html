Checking in with your Crypts

Last updated: 1st June 2016
  • Using The Official Client
  • Using CURL
  • Other Clients

Once you have successfully created a crypt then checking in (aka resetting the deadmans countdown) is acheived with the checkin command.

Recommended Usage

You should configure a scheduler such as cron to regularly checkin so as to reset the deadline countdown. Ideally you would checkin as regularly as your checkin duration rather than the cumulative duration for two reasons;

  1. In future versions we may send notifications if your crypt has missed a checkin
  2. It guarantees that a single missed event (rebooting your PC / clock skew etc) won't result in you missing the deadline entirely.

Checking in is easy;

rcrypt checkin -crypt=0ad1f05ccfc3a01327225d9635263152

You will need to generate and decrypt a challenge before you can use a CURL request to checkin but once generated and decrypted a checkin is as simple as;

curl -v -H "Accept: application/json" -X POST -d "{\"challenge\":\"5f185b74b96edab86b0c29f26760c433\",\"user_id\":9, \"challenge_id\":22}" https://ripacrypt.download/1/crypt/0ad1f05ccfc3a01327225d9635263152/

There are no 3rd party clients we are aware of. If this changes please let us know.
