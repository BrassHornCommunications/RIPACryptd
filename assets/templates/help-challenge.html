Generating a Challenge

Last updated: 1st June 2016
  • Using The Official Client
  • Using CURL
  • Other Clients

RIPACrypt uses challenges that are encrypted with your GPG public key as a way of proving that you are authorised to make changes to your account.

The RIPACrypt client transparently requests and decrypts challenges before performing other actions.

Recommended Usage

We do not recommend manually requesting a challenge as it is rarely part of the normal application usage to require an out of band challenge.

rcrypt getchallenge

Acquiring a challenge requires sending your User ID and corresponding GPG fingerprint.

curl -v -H "Accept: application/json" -X POST -d "{\"fingerprint\":\"447055F36F6D60C1\",\"user_id\":9}" https://ripacrypt.download/1/challenge/

There are no 3rd party clients we are aware of. If this changes please let us know.
